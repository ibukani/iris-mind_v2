# Iris Runtime API

This document describes the external gRPC contract for `iris-mind_v2`. Clients like `iris-cli_v2` should use this API to interact with the cognitive runtime.

## Runtime Service Endpoint

The default `IrisRuntimeService` is a standard gRPC server. The runtime operates over unary RPCs for simple interactions, specifically `SubmitObservation` and `GetRuntimeInfo`.

## RPC Methods

### `GetRuntimeInfo`

Returns stable metadata about the connected Mind instance.
Clients can use this upon connecting to ensure compatibility.

```protobuf
message GetRuntimeInfoResponse {
  string runtime_name = 1;         // e.g. "iris-mind"
  string runtime_version = 2;      // e.g. "0.1.0"
  string api_version = 3;          // e.g. "iris.runtime.v1"
  repeated string supported_features = 4; // e.g. "submit_observation", "persistent_account", "ephemeral_space"
}
```

### `SubmitObservation`

Submits an event from the client side into the cognitive runtime. For simple CLI chat, this is used to send user messages.

```protobuf
message SubmitObservationRequest {
  string correlation_id = 1;
  iris.api.v1.Observation observation = 2;
}
```

## Required and Recommended Fields for CLI Clients

### `SubmitObservationRequest`
- **`correlation_id`**: A client-generated unique ID for tracking the request. It is echoed back in the response.

### `Observation`
- **`observation_id`**: A client-generated unique ID for the observation.
- **`session_id`**: A client-generated session identifier. For one-shot CLI commands this can be unique per command, but for REPLs, keep it stable across the session.
- **`kind`**: Should be `OBSERVATION_KIND_ACTOR_MESSAGE`.
- **`occurred_at`**: Must be provided. A protobuf `Timestamp` of when the event happened.

### `ObservationContext`
- **`source`**: Should be `"cli"`.
- **`account_ref`** *(Recommended)*: External clients should identify themselves via `account_ref`, **not** `account_id` or `Identity`.
- **`space_ref`** *(Recommended)*: Contextual space representation. Resolved ephemerally.

### `ExternalAccountRef`
Used by the server's `AccountIdentityResolver` to lookup or implicitly create an internal Identity.
- **`provider`**: Should be `"cli"`.
- **`provider_subject`**: A stable identifier for the local user on the client side (e.g., local OS username or fixed CLI ID).
- **`display_name`**: Required by the mapper. The user's visible name.
- **`actor_kind`**: Should be `ACTOR_KIND_HUMAN`.

### `ExternalSpaceRef`
Represents an external interaction context. It is resolved ephemerally on the server side and is not persisted.
- Optional for one-shot CLI prompts.
- Recommended for CLI REPL loops (e.g., `provider_space_ref="session:..."`).

## Example CLI Request

```python
import time
from iris.generated.iris.runtime.v1 import runtime_pb2
from iris.generated.iris.api.v1 import observations_pb2, identity_pb2, spaces_pb2
from google.protobuf.timestamp_pb2 import Timestamp

ts = Timestamp()
ts.GetCurrentTime()

request = runtime_pb2.SubmitObservationRequest(
    correlation_id="cli-req-123",
    observation=observations_pb2.Observation(
        observation_id="obs-123",
        session_id="repl-456",
        kind=observations_pb2.OBSERVATION_KIND_ACTOR_MESSAGE,
        occurred_at=ts,
        context=observations_pb2.ObservationContext(
            source="cli",
            account_ref=identity_pb2.ExternalAccountRef(
                provider="cli",
                provider_subject="local-user-id",
                display_name="Local User",
                actor_kind=identity_pb2.ACTOR_KIND_HUMAN,
            ),
            space_ref=spaces_pb2.ExternalSpaceRef(
                provider="cli",
                provider_space_ref="session:repl-456",
                display_name="CLI REPL Session",
                space_kind=spaces_pb2.SPACE_KIND_DIRECT_MESSAGE,
            )
        ),
        actor_message=observations_pb2.ActorMessagePayload(
            text="Hello, Iris!",
            external_message_id="msg-123"
        )
    )
)
```

## `PresentedOutput`

The response from `SubmitObservation` contains a `PresentedOutput`.
- **`text`**: The main user-visible response generated by the cognitive runtime. This is what CLI clients should print.
- **`style_hint`, `emotion_hint`, `expression_hint`**: Optional UI presentation hints.
- **`delay_ms`, `priority`, `interruptible`**: Presentation timing and behavioral hints. Simple CLI clients may ignore these initially.
